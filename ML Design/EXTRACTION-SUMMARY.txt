KEY FRAMEWORKS EXTRACTED
========================

1. DELIVERY FRAMEWORK (45-minute interview structure)
   Problem Framing (5-7) → High-level Design (2-3) → Data/Features (10) → 
   Modeling (10) → Inference/Evaluation (7) → Deep Dives (remaining)

2. FIVE-LAYER EVALUATION STACK (applies to every problem)
   Business Objective → Product Metrics → ML Metrics → 
   Evaluation Methodology → Address Challenges

3. GENERALIZATION FAILURE MODES
   Overfitting (memorize) ← vs → Underfitting (too simple)
   Data Drift (world changed)
   
4. REGULARIZATION TECHNIQUES
   Dropout, Layer Normalization, L2 (Ridge), L1 (Lasso), Early Stopping

5. PROBLEM-TYPE SPECIFIC STRATEGIES
   ├─ Classification: Focus on Precision/Recall trade-off, PR-AUC for imbalance
   ├─ Recommendations: NDCG, Coverage, exploration vs exploitation
   ├─ Search: NDCG@k, handle query ambiguity, long-tail, position bias
   └─ Generative: Portfolio of metrics, human validation, hallucination tracking

================================================================================
CROSS-CUTTING PATTERNS (Apply to ALL problems)
==============================================

1. Business Objective drives everything
2. Evaluation is multi-layer, multi-metric
3. Simple baseline before complex model
4. Data matters more than model
5. Constraints (scale, latency, cost) drive design
6. Inference is production reality
7. Monitor or die (drift detection essential)
8. Feedback loops everywhere (classification, recommendations, search, generative)
9. Edge cases are features (cold-start, long-tail, distribution shift)
10. Communication = success (clear thinking > perfect design)
