# DQN From Scratch — Building a Game-Playing Brain

> Here is a puzzle: in 2013, a small team at DeepMind fed raw Atari game pixels into a neural network and told it nothing about the rules. No instructions. No strategy guide. Just the pixels on screen and the score. The network learned to play Breakout, Pong, and Space Invaders — some at superhuman level. The algorithm behind this was called DQN (Deep Q-Network), and it combined three ideas that had each existed for years but had never worked together. This notebook builds DQN from scratch so you can see exactly how those three ideas fit.

## The analogy: studying for a huge exam

Imagine you are preparing for a test that covers thousands of topics. You have three problems:

**Problem 1 — Too many flashcards.** If you tried to make one flashcard for every possible question, you would need millions of cards. Instead, you learn *patterns*: "Questions about the French Revolution usually involve these key dates and causes." You can now answer questions you have never seen before, because you understand the pattern, not just individual facts.

**Problem 2 — Studying in order is bad.** If you read Chapter 1 twenty times in a row, then Chapter 2 twenty times, you forget Chapter 1 by the time you finish Chapter 2. A better strategy: write each key idea on a separate card, shuffle the cards, and study them in random order. This way every study session covers a mix of topics.

**Problem 3 — The answer key keeps changing.** Imagine the test answers are generated by your own knowledge. As you learn more, the "correct" answers shift. You are trying to match a target that moves every time you study. The fix: take a snapshot of the answer key and freeze it. Study against that snapshot for a while. Then take a new snapshot. The target stays still long enough for you to catch up.

DQN solves exactly these three problems:
1. A **neural network** replaces the flashcard collection (Q-table), learning patterns instead of memorizing.
2. **Experience replay** shuffles past experiences so training data is diverse, not sequential.
3. A **target network** freezes the learning target periodically so it does not move while the agent is updating.

## What the analogy gets right

The core parallel is exact. The neural network generalizes like pattern-based studying. Experience replay breaks correlation like shuffling flashcards. The target network stabilizes learning like freezing the answer key. Each innovation addresses a specific failure mode that prevented earlier attempts from working.

## The concept in plain words

DQN is Q-learning where the Q-table is replaced by a neural network. The network takes a state (for example, the four numbers describing a cart and pole) and outputs one Q-value for each possible action. The agent picks the action with the highest Q-value, just like tabular Q-learning.

Training works the same way too: compute a target (reward plus the discounted best future Q-value), measure how far the network's prediction is from that target, and use gradient descent to adjust the weights. The loss function is the squared distance between the prediction and the target.

But naive Q-learning with a neural network is unstable. Three things go wrong:

1. **Correlated data.** Consecutive experiences look almost identical (the cart moved 0.01 to the right, then 0.01 more). Training on correlated data causes the network to overfit to recent experiences and forget older ones. Experience replay fixes this by storing thousands of past experiences in a buffer and sampling random mini-batches for training.

2. **Moving target.** The target r + γ max Q(s', a') uses the same network that is being updated. Every weight update changes both the prediction and the target simultaneously. This is like trying to hit a dartboard that shifts every time you throw. The target network fixes this by keeping a frozen copy of the network. Targets are computed using the frozen copy, which only updates every N steps.

3. **Bootstrapping with function approximation.** The combination of using estimates to update estimates (bootstrapping) with a neural network (function approximation) and off-policy data (experience replay) is the deadly triad. The experience replay and target network together tame this instability enough for learning to succeed.

## Where the analogy breaks down

In real studying, forgetting Chapter 1 while learning Chapter 2 is gradual. In a neural network, updating weights for one state can abruptly change Q-values for completely different states, because all states share the same parameters. This "catastrophic interference" is more sudden and harder to control than human forgetting.

## The three innovations, one more time

It helps to see how each piece solves a specific problem:

| Problem | What goes wrong | DQN's fix |
|---------|----------------|-----------|
| Table too large | Cannot store Q-values for millions of states | Neural network learns patterns, generalizes to new states |
| Correlated training data | Network overfits to recent experiences | Experience replay samples random mini-batches from a buffer |
| Moving target | Updating the network changes the target it is chasing | Target network freezes the target, updates every N steps |

Remove any one of these three innovations and DQN fails. Mnih et al. (2015) showed this experimentally: without replay, performance drops on most Atari games; without the target network, training becomes unstable.

## Victory lap

You just understood the algorithm that started the deep reinforcement learning revolution. DQN was the first method to learn control policies directly from high-dimensional sensory input (raw pixels) at a level competitive with human players. Every modern deep RL algorithm — Double DQN, Dueling DQN, Rainbow, and the entire DQN family — builds directly on these three ideas. When someone says "deep reinforcement learning," they are talking about the lineage that begins here.
